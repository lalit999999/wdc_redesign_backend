WDC BACKEND → FRONTEND INTEGRATION HANDOFF
=========================================

Backend Base URL
----------------
Use this in frontend:
- http://localhost:5000/api

Auth alias also exists:
- http://localhost:5000/auth


1) AUTH FLOW (GOOGLE OAUTH)
---------------------------
Frontend login button should redirect browser to:
- http://localhost:5000/api/auth/google
  (or /auth/google)

After successful login, backend returns JSON from callback with:
- token (JWT)
- user object

Store token on frontend (localStorage/sessionStorage) and send in every protected request:
Authorization: Bearer <token>


2) REQUIRED REQUEST HEADERS
---------------------------
For protected routes:
- Authorization: Bearer <jwt_token>

For JSON body routes:
- Content-Type: application/json


3) COMMON RESPONSE FORMAT
-------------------------
Most responses return:
{
  "success": true|false,
  "message": "...",
  ...data
}


4) PUBLIC ROUTES
----------------
GET /
Response 200:
{ "message": "Backend is running" }

GET /health
Response 200:
{ "status": "ok", "message": "WDC Server is running" }


5) AUTH ROUTES
--------------
GET /api/auth/google
- Starts Google OAuth flow (browser redirect)

GET /api/auth/google/callback
Response 200:
{
  "success": true,
  "message": "Google authentication successful",
  "token": "<jwt>",
  "user": { ... }
}

GET /api/auth/google/failure
Response 401:
{ "success": false, "message": "Google authentication failed" }


6) USER ROUTES (PROTECTED)
--------------------------
GET /api/users/profile
Response 200:
{ "success": true, "user": { ... } }

PUT /api/users/profile
Body (any subset allowed):
{
  "fullName": "string",
  "mobileNumber": "string",
  "githubLink": "string",
  "linkedinLink": "string",
  "resumeLink": "string",
  "about": "string"
}
Response 200:
{
  "success": true,
  "message": "Profile updated successfully",
  "user": { ... , "profileCompletion": 0-100 }
}

GET /api/users/application-status
Response 200:
{
  "success": true,
  "applicationStatus": "pending | shortlisted | rejected"
}

ADMIN ONLY:
GET /api/users
Response 200:
{ "success": true, "users": [ ... ] }

PATCH /api/users/:id/approve
Response 200:
{
  "success": true,
  "message": "User approved successfully",
  "user": { ... , "applicationStatus": "shortlisted" }
}

PATCH /api/users/:id/reject
Response 200:
{
  "success": true,
  "message": "User rejected successfully",
  "user": { ... , "applicationStatus": "rejected" }
}


7) ASSIGNMENT ROUTES
--------------------
ADMIN ONLY:
POST /api/assignments
Body:
{
  "title": "string",
  "description": "string",
  "category": "string",
  "deadline": "ISO date string"
}
Response 201:
{
  "success": true,
  "message": "Assignment created successfully",
  "assignment": { ... }
}

DELETE /api/assignments/:id
Response 200:
{ "success": true, "message": "Assignment deleted successfully" }

SHORTLISTED STUDENT ONLY:
GET /api/assignments
Response 200:
{ "success": true, "assignments": [ ... ] }

POST /api/assignments/:id/submit
Body:
{
  "githubLink": "string",
  "hostedLink": "string",
  "finalResumeLink": "string",
  "remark": "string"
}
Response 201:
{
  "success": true,
  "message": "Assignment submitted successfully",
  "submission": { ... }
}

}
Response 201:

ALL AUTHENTICATED USERS:
GET /api/announcements
Response 200:
{ "success": true, "announcements": [ ... ] }


9) IMPORTANT ERROR RESPONSES (HANDLE IN FRONTEND)
--------------------------------------------------
401 Missing/Bad header:
{ "success": false, "message": "Access token missing or invalid format" }

401 Invalid/Expired token:
{ "success": false, "message": "Invalid or expired token" }

403 Admin-only route:
{ "success": false, "message": "Access denied: admin only" }

403 Non-shortlisted assignment access:
{ "success": false, "message": "Access denied" }

400 Late submission:
{ "success": false, "message": "Submission deadline has passed" }

409 Duplicate submission:
{ "success": false, "message": "Assignment already submitted" }

404 Unknown route:
{ "success": false, "message": "Route not found: <path>" }
